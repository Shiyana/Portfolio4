Artifact 4: Clustering Analysis with K-Means

Overview

This artifact focuses on my implementation of the K-Means clustering algorithm in Python to solve unsupervised learning tasks. Central to this artifact was a guided class chatbot activity, where I explored and responded to three real-world machine learning challenges—handling missing data, addressing noisy data, and managing class imbalance.

In the first scenario, I worked with a dataset containing missing values, a common issue in healthcare and business analytics. I applied various strategies like mean/mode imputation, KNN imputation, and complete-case analysis. Each technique had its merits, and I learned the importance of data context in selecting the right method.

In the second scenario—noisy data—I explored smoothing techniques such as moving averages and exponential smoothing, alongside outlier detection using IQR and z-scores. I realized the impact of noise on clustering algorithms, especially in skewing centroids and cluster separations.

The third scenario—class imbalance—challenged me to use oversampling (SMOTE) and undersampling techniques to balance skewed class distributions. Through this, I discovered how models could be biased if the data is not fairly represented. Integrating these lessons into clustering further enhanced my approach to identifying reliable, interpretable clusters.

These scenarios provided a bridge between theoretical learning and practical application, pushing me to think critically and build intuition for working with real-world data challenges. Combined with the technical implementation of K-Means, this exercise strengthened my skills in exploratory data analysis, data cleaning, and ethical ML practices.

Process

Detailed steps followed:

  1. Data Preprocessing: Cleaned the dataset by addressing missing values with imputation strategies and scaled features using StandardScaler to normalize the range of values.
  2. Exploratory Analysis: Used pairplots and correlation heatmaps to understand data distributions and relationships.
  3. Cluster Tendency Assessment: Employed the Elbow Method and Silhouette Score to determine the optimal number of clusters (k=3).
  4. Model Application: Implemented K-Means using scikit-learn and visualized the output using matplotlib and seaborn for 2D plotting.
  5. Validation: Assessed model performance through average Silhouette Coefficient and interpretability of centroids.

Challenges & Solutions

  • Scenario 1 - Missing Data: Choosing between deletion vs. imputation required balancing completeness with potential bias. KNN imputation preserved variance better.
  • Scenario 2 - Noisy Data: High variance in readings introduced noise into clusters. Applied moving average smoothing and removed extreme outliers using IQR.
  • Scenario 3 - Class Imbalance: Though K-Means is unsupervised, it is affected by imbalanced representations. Synthetic techniques like SMOTE and stratified sampling ensured better groupings post-preprocessing.
  • Cluster Interpretation: Initially, centroid values were hard to label. Post-clustering, used mean feature values to assign meaningful labels like "High Income-Low Spending".

Key Takeaways

  • Data quality (noise, completeness, balance) significantly impacts unsupervised learning outcomes.
  • Visualization and domain understanding are critical for validating clustering results.
  • Ethical considerations, especially around fairness and representation, must guide preprocessing and modeling decisions.
  • Refined understanding of practical techniques: Elbow vs. Silhouette, preprocessing pipelines, and scaling techniques.
  • Soft skills like critical thinking, scenario-based troubleshooting, and ethical reasoning are integral to successful AI practice.
