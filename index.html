Overview

Project Summary: This artifact demonstrates a comprehensive clustering analysis using K-Means. It combines technical execution, leadership reflection, and ethical considerations in AI. Throughout the project, I tackled three major scenarios in ML: missing data, noisy data, and class imbalance.

Scenario Explanations:

  • Scenario 1 - Missing Data: I explored how data absence affects model outcomes and compared strategies like mean imputation and KNN imputation using Python’s `sklearn.impute`.
  • Scenario 2 - Noisy Data: I tested smoothing techniques including rolling average, Savitzky-Golay filter, and visualization of residual noise using matplotlib and seaborn.
  • Scenario 3 - Class Imbalance: I implemented SMOTE (Synthetic Minority Over-sampling Technique) and evaluated results with precision, recall, and F1 scores.

Process

Step-by-step Execution:

  • Step 1: Data acquisition and inspection with `pandas`.
  • Step 2: Preprocessing with standardization and outlier handling using `StandardScaler` and z-scores.
  • Step 3: Feature selection and visualization using `matplotlib`, `seaborn` heatmaps.
  • Step 4: Optimal cluster selection using Elbow and Silhouette analysis.
  • Step 5: KMeans clustering with interpretation of centroids and cluster distribution.

Challenges & Solutions

  • Dealing with mixed data types: I encoded categorical values using one-hot encoding to ensure compatibility with clustering algorithms.
  • Cluster interpretability: I plotted cluster means and variances to understand key attributes in each segment.
  • Validating model performance: Beyond silhouette score, I used Davies-Bouldin Index to confirm compactness and separation.
  • Bias risks: I conducted subgroup analysis to identify potential demographic skews in cluster distribution.
